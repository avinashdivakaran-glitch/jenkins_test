tnn_server/wifi/adapter
{
    "HeaderID": "12345",
    "timestamp": "2025-10-30T14:45:00Z",
    "SetPower": true,
    "data": {
        "IsAvailable": true,
        "Adapter": "AdapterA",
        "IsPowered": true,
        "Error": ""
    }
}
Topic : Control and Display wifi adapter state
Process :
    "SetPower" use for power on the wifi adapter. true -> On. false -> Off
    "IsAvailable" Display true if the backend able to detect the wifi adapter
    "Adapter" Display the available adapter name
    "IsPowered" Display true if the adapter is power on. else display off
    "Error" If there is any error in process display it
when "SetPower" is set true backend start the process, if the process is complete "IsPowered" set to true. else "IsPowered" is false and "Error" will display

    "SetScan": true,
    "data": {
        "IsScan": true,
        "Error": ""
    }
}
Topic : Control and Display wifi Scan process
Process : 
    "SetScan" use for trigger backend wifi scan function. if true start the scan. else stop the scan
    "IsScan" display true if the start scan method complete without error
    "Error" if there is any problem in process display error

tnn_server/wifi/func/connect
{
    "HeaderID": "CON-5531",
    "timestamp": "2025-10-30T14:45:00Z",
    "SetConnect": true,
    "data": {
        "IsConnected": true,
        "SSID": "OfficeWiFi",
        "Password": "office@2024",
        "Recon": false,
        "Error": ""
    }
}
Topic : Control and Display wifi device connect process
Process :
    "SetConnect" use for trigger backend wifi connecting method.
    "IsConnected" Display true if the wifi is connected to given device
    "SSID" ssid of the wifi AP 
    "Password" password of the wifi AP 
    "Recon" set true if need to auto reconnect, else false
    "Error" if there is any problem in process display error
when "SetConnect" set to true backend try to connect wifi with properties with "SSID", "Password" and "Recon". if device connected to wifi "IsConnected" set to true. else false.
If there is any problem "Error" with update


tnn_server/wifi/func/disconnect
{
    "HeaderID": "DISC-777",
    "timestamp": "2025-10-30T14:45:00Z",
    "SetDisconnect": true,
    "SetRemove": false,
    "data": {
        "IsDisconnected": true,
        "SSID": "OfficeWiFi",
        "Error": ""
    }
}
Topic : Control and Display device disconnect process
Process : 
    "SetDisconnect" use for trigger backend wifi disconnect method
    "SetRemove" use for trigger backend wifi forget method
    "IsDisconnected" true if given device is disconnect or remove successfully
    "SSID" ssid of the device that need to disconnect or remove
    "Error" if there is error in process error with update
when need to disconnect the connected wifi device "IsDisconnected" set to true with "SSID". if the process complete successfully "IsDisconnected" set to true, else set as false and "Error" value display
Same method for forget a device
